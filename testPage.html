Event Bus Refactor — Strongly Typed Events Only

Branch: eventBus-strongly-typed-events-only

Goal: Remove SubscribeAll, EventEnvelope, and EventBusHelper from the event bus system. Replace all usages with direct _eventBus.Publish(...) calls using strongly typed events. EventReason is kept and published directly rather than wrapped.

Affected Files Overview

File Action Summary
Shared/EventBus/EventEnvelope.cs DELETE EnvelopeId never read anywhere. MsgType redundant. CreatedAt only used in dead loggers. Entire class is a wrapper with no remaining purpose.
Shared/EventBus/EventBusHelper.cs DELETE Generic overload is a pure pass-through with no added value. EventReason overload only existed to wrap in EventEnvelope. Both become unnecessary.
Shared/EventBus/EventReason.cs KEEP Unchanged. Published directly on the bus instead of being wrapped in EventEnvelope.
Shared/EventBus/IEventBus.cs UPDATE Remove SubscribeAll method declaration.
Shared/EventBus/BasicEventBus.cs UPDATE Remove SubscribeAll method, _globalSubscriptionsList field, and the foreach loop over global subscribers inside Publish<T>.
Shared/EventBus/DomainEvents/NetworkEvents.cs ADD Add new PacketTransmittedEvent record with fields: Guid MessageId, string MessageType, string Direction, ConnectionId? Client, TransportEnvelope Envelope, DateTimeOffset Timestamp. Replaces the anonymous object hack used on the PacketLog channel.
Shared/Logging/EventBusLogger.cs UPDATE Replace bus.SubscribeAll(HandleLog) with explicit Subscribe<EventReason> calls per relevant categories. Update HandleLog to accept EventReason directly, removing the double-cast through EventEnvelope.
Shared/Logging/FileLogger.cs UPDATE Same pattern as EventBusLogger — replace SubscribeAll with per-category Subscribe<EventReason>. Update handler to accept EventReason directly.
Shared/Logging/PacketLogger.cs UPDATE Replace Subscribe<EventEnvelope>(PacketLog) with Subscribe<PacketTransmittedEvent>(PacketLog). Rewrite OnPacketEvent to use typed fields directly, removing the Payload cast and anonymous object.
Server.GUI/MainWindow.xaml.cs UPDATE Replace 4x Subscribe<EventEnvelope>(category) with Subscribe<EventReason>(category). Update the 4 handler methods to accept EventReason directly. Delete the dead OnAnyEventReceived method (leftover from original SubscribeAll era).
Client.GUI/MainWindow.xaml.cs UPDATE Replace Subscribe<EventEnvelope>(PacketLog) with Subscribe<PacketTransmittedEvent>(PacketLog). Delete OnPacketReceived and OnPacketReceivedAsync. Replace with a typed handler — eliminates reflection on the anonymous object (dataType.GetProperty("Direction")).
Server.Core/Network/Supervisor/StandardNetworkSupervisor.cs UPDATE ~20 call sites. Replace all EventBusHelper.PublishEvent(_eventBus, category, new EventReason(...)) with _eventBus.Publish(category, new EventReason(...)). Replace PacketLog anonymous-object calls with new PacketTransmittedEvent(...).
Server.Core/CommandPipeline/CommandPipelineOrchestrator.cs UPDATE Replace all EventBusHelper.PublishEvent(...) with _eventBus.Publish(...).
Server.Core/CommandPipeline/Authentication/StandardAuthenticationPipeline.cs UPDATE Replace all EventBusHelper.PublishEvent(...) (both overloads present) with _eventBus.Publish(...).
Server.Core/Persistence/InMemoryAuthenticationService.cs UPDATE Replace all EventBusHelper.PublishEvent(...) with _eventBus.Publish(...).

Server.Core/Domain/Services/ChatService/ChatService.cs UPDATE Replace EventBusHelper.PublishEvent(...) with _eventBus.Publish(...).
Client.Core/Network/ClientNetworkService.cs UPDATE Replace EventBusHelper.PublishEvent(...) with _eventBus.Publish(...). Replace PacketLog anonymous-object calls with new PacketTransmittedEvent(...).
